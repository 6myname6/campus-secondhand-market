# 二手交易平台 — 使用手册

## 1. 环境准备

### 1.1 系统要求
| 项目 | 要求 |
|------|------|
| 操作系统 | Windows / macOS / Linux |
| Java | JDK 17 或更高版本 |
| 数据库 | MySQL 8.0 |
| 内存 | 建议 4GB 以上（同时运行服务端+客户端） |

### 1.2 数据库初始化

1. 确保 MySQL 服务已启动
2. 使用 root 用户登录 MySQL，执行初始化脚本：

```bash
mysql -u root -p < init.sql
```

3. 如需修改数据库连接信息，编辑 `server/src/main/resources/db.properties`：

```properties
db.driver=com.mysql.cj.jdbc.Driver
db.url=jdbc:mysql://localhost:3306/second_hand_trading?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
db.user=root
db.password=123456
```

> 也可通过环境变量 `DB_URL`、`DB_USER`、`DB_PASS` 覆盖配置。

### 1.3 编译打包

在项目根目录执行：

```bash
mvn clean package -DskipTests
```

打包产物位于：
- `server/target/server-jar-with-dependencies.jar`
- `client/target/client-jar-with-dependencies.jar`

`dist/` 目录中已包含预编译好的 JAR 文件。

## 2. 启动应用

### 2.1 启动服务端

**方式一：使用批处理脚本（Windows）**
双击 `dist/启动服务端.bat`

**方式二：命令行**
```bash
java -jar dist/server.jar
```

启动成功后会看到类似输出：
```
Netty server started on port 8081
```

> 服务端默认监听 **8081** 端口，请确保该端口未被占用。

### 2.2 启动客户端

**方式一：使用批处理脚本（Windows）**
双击 `dist/启动客户端.bat`

**方式二：命令行**
```bash
java -jar dist/client.jar [主机地址] [端口号]
```

默认连接 `127.0.0.1:8081`。

### 2.3 启动双客户端（测试聊天/交易）

双击 `dist/启动双客户端.bat`，会同时启动两个客户端窗口，方便测试聊天和交易功能。

## 3. 功能操作指南

### 3.1 注册与登录

**注册：**
1. 启动客户端后，进入登录界面
2. 切换到"注册"标签
3. 填写用户名、密码、手机号（可选）、邮箱（可选）
4. 点击"注册"按钮
5. 注册成功后，手动切换到"登录"标签

**登录：**
1. 输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后自动跳转到首页

> 默认连接配置在界面中可修改，连接远程服务器时输入对应 IP 地址。

### 3.2 浏览商品

**首页浏览：**
- 登录后默认进入首页，显示所有在售商品
- 使用顶部分类下拉菜单按分类筛选
- 使用搜索框按标题关键词搜索
- 底部"上一页/下一页"按钮翻页

**商品卡片信息：**
- 商品缩略图
- 商品标题
- 价格（红色显示）
- 浏览次数

### 3.3 查看商品详情

点击任意商品卡片进入详情页，可以看到：
- 多张商品图片（可浏览）
- 商品标题、售价、原价
- 商品状态（在售/已售）
- 浏览次数和发布时间
- 商品详细描述

**详情页操作：**
- **收藏**：点击红心按钮收藏/取消收藏
- **立即购买**：弹出购买窗口，填写收件人姓名、电话、地址后确认下单
- **联系卖家**：自动创建聊天会话并跳转到聊天页面

### 3.4 发布商品

1. 点击导航栏"发布"按钮
2. 填写商品信息：
   - **标题**：商品名称
   - **分类**：选择合适的商品分类
   - **售价**：销售价格（元）
   - **原价**：原始价格（可选）
   - **描述**：商品详细描述
3. 点击"选择图片"添加商品图片（支持多选）
4. 点击"发布"按钮完成发布
5. 发布成功后可在首页看到自己的商品

> 图片会在本地选择后以 Base64 编码上传到服务端。

### 3.5 订单管理

**查看订单：**
点击导航栏"我的订单"进入订单列表。

**订单状态与操作：**

| 状态 | 显示颜色 | 可用操作 | 说明 |
|------|----------|----------|------|
| 待支付 | 红色 | 付款 / 取消 | 刚下单，需尽快付款 |
| 待发货 | 橙色 | —（等待卖家操作） | 已付款，等待卖家发货 |
| 待收货 | 蓝色 | 确认收货 | 卖家已发货，收到货后确认 |
| 已完成 | 绿色 | — | 交易完成 |
| 已取消 | 灰色 | — | 订单已取消 |

**买家操作流程：**
1. 下单 → 订单状态：**待支付**
2. 点击"付款" → 状态变为：**待发货**
3. 等待卖家发货 → 状态变为：**待收货**
4. 收到商品后点击"确认收货" → 状态变为：**已完成**

**卖家操作流程：**
1. 收到订单通知（在订单列表中查看已售商品相关订单）
2. 买家付款后，点击订单进入详情
3. 填写物流单号，点击"发货" → 状态变为：**待收货**

**取消订单：**
- 在非终态（未完成/未取消）的订单中点击"取消"
- 需填写取消原因

### 3.6 收藏功能

**添加收藏：**
- 在商品详情页点击心形图标即可收藏

**查看收藏：**
- 点击导航栏"收藏"按钮查看所有收藏商品
- 点击收藏商品可进入详情页

### 3.7 聊天功能

**发起聊天：**
- 在商品详情页点击"联系卖家"
- 系统自动创建（或打开已有的）聊天会话

**聊天界面：**
- 左侧灰色气泡：对方的消息
- 右侧蓝色气泡：自己的消息
- 底部输入框：输入消息后按回车或点击"发送"
- 消息每 3 秒自动刷新（轮询新消息）

**查看聊天列表：**
- 点击导航栏"聊天"按钮
- 显示所有聊天会话（对方用户名、最后消息、时间）
- 有未读消息的会话会显示红色角标
- 点击会话进入聊天界面

### 3.8 个人中心

点击导航栏"我的"进入个人中心：
- 显示当前用户名
- 可编辑手机号和邮箱
- 点击"保存"更新个人信息
- 点击"退出登录"返回登录界面

## 4. 常见问题

### Q: 启动服务端报"Address already in use"？
A: 8081 端口已被占用。关闭占用该端口的程序，或修改 `NettyServer.java` 中的端口号。

### Q: 客户端无法连接服务端？
A: 检查：
1. 服务端是否已启动
2. 连接地址是否正确（默认 127.0.0.1:8081）
3. 防火墙是否允许 8081 端口通信

### Q: 数据库连接失败？
A: 检查：
1. MySQL 服务是否运行
2. `db.properties` 中用户名密码是否正确
3. 数据库 `second_hand_trading` 是否已创建（执行 init.sql）

### Q: 图片无法显示？
A: 检查 `uploads/` 目录是否存在且有写入权限。图片通过 Base64 编码传输，确保网络稳定。

### Q: 聊天消息不即时显示？
A: 聊天采用 3 秒轮询机制，不是真正的实时推送。消息会在 3 秒内显示，属于正常现象。

## 5. 项目运行演示流程

### 完整交易演示（约 5 分钟）

**准备工作：**
启动一个服务端 + 两个客户端（用"启动双客户端"脚本）。

**用户A（卖家）：**
1. 注册账号 `alice`，登录
2. 点击"发布"，填写商品信息，选择图片，发布
3. 等待买家下单

**用户B（买家）：**
1. 注册账号 `bob`，登录
2. 在首页看到 alice 发布的商品
3. 点击进入详情，点击"立即购买"
4. 填写收件信息，确认下单
5. 在"我的订单"中找到订单，点击"付款"

**用户A（回到卖家）：**
6. 在"我的订单"中找到待发货订单
7. 进入详情，填写物流单号，点击"发货"

**用户B（回到买家）：**
8. "我的订单"中看到状态已变为"待收货"
9. 点击"确认收货"完成交易

**聊天测试：**
10. B 在商品详情页点击"联系卖家"
11. A 在"聊天"中看到未读消息
12. 双方互相发送消息测试聊天功能
