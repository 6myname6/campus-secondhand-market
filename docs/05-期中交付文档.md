## 一、项目概述

### 1.1 项目背景

随着高校校园规模的扩大和学生人数的增长，校内二手物品交易需求日益旺盛。传统的QQ群、贴吧等交易方式存在信息杂乱、缺乏交易流程保障、沟通效率低等问题。为此，我们开发了一套 **C/S 架构的校园二手交易平台**，为学生提供安全、便捷、一站式的二手物品交易服务。

### 1.2 项目目标

- 提供用户注册登录与个人信息管理
- 支持商品发布、分类浏览、关键词搜索
- 实现完整的订单生命周期管理（下单→支付→发货→收货→完成）
- 支持商品收藏功能
- 提供买卖双方的即时聊天功能

### 1.3 技术选型

| 层级 | 技术方案 | 选型理由 |
|------|----------|----------|
| 构建工具 | Maven 多模块 | 服务端/客户端分离，各自独立打包 |
| 开发语言 | Java 17 | LTS 版本，生态成熟 |
| 服务端框架 | Netty 4.1.112 | 高性能 NIO 网络框架，适合自定义通信协议 |
| 客户端框架 | Java Swing | JDK 自带 GUI 框架，无需额外依赖 |
| 数据库 | MySQL 8.0 | 开源关系型数据库，事务支持完善 |
| 连接池 | HikariCP 5.0.1 | 业界性能最优的 JDBC 连接池 |
| 序列化 | Gson 2.10.1 | 轻量级 JSON 库，API 简洁 |
| 日志 | SLF4J + Logback | 高性能日志框架，支持滚动归档 |
| 测试 | JUnit 5 | Java 主流单元测试框架 |

### 3.3 通信协议

- **协议类型**：JSON over TCP，换行符 `\n` 分隔消息帧
- **请求格式**：`{ "action": "xxx", "params": {...}, "_reqId": N }`
- **响应格式**：`{ "success": true/false, "data": {...}, "message": "..." }`
- **认证方式**：需认证的 action 在 params 中携带 `token` 字段
- **路由数量**：共 30 个 action，涵盖用户/商品/订单/收藏/聊天/文件 6 大模块

## 四、项目亮点

### 4.1 技术亮点

1. **自研轻量 ORM**：基于 Java 反射 + 自定义 `@Table`/`@Column` 注解，实现自动驼峰↔蛇形映射、通用 CRUD、PreparedStatement 防 SQL 注入，深入理解 ORM 底层原理。

2. **函数式路由分发**：`ServerHandler` 使用 `Map<String, Function<JsonObject, Response>>` 实现灵活的路由注册；`requireAuth` 高阶函数包装器在不侵入业务代码的前提下注入 Token 校验逻辑。

3. **事务模板模式**：`BaseService.execTx` 使用 Lambda 回调实现"事务包围"，业务代码只需编写 SQL 逻辑，连接的获取/提交/回滚/释放由框架统一管理。

4. **安全防护**：SQL 参数化查询防注入、聊天消息 HTML 转义防 XSS、文件路径规范化防穿越、参数校验器防恶意输入。

5. **C/S 全链路实现**：从数据库设计到 Netty 自定义协议到 Swing 图形界面到 fat JAR 打包部署，覆盖完整的 C/S 架构开发流程。

### 4.2 工程亮点

1. **Maven 多模块管理**：server 和 client 模块分离，共享实体类，各自独立打包为 fat JAR。
2. **环境变量配置**：数据库连接支持环境变量覆盖（`DB_URL`/`DB_USER`/`DB_PASS`），适配容器化部署。
3. **完整的订单状态机**：待支付→待发货→待收货→已完成，含取消操作，状态转变有明确的前置条件校验。
4. **双客户端测试支持**：提供一键启动双客户端脚本，方便演示和测试聊天/交易交互流程。

## 五、项目不足与后续计划

### 5.1 已知不足

| 分类 | 问题 | 影响 |
|------|------|------|
| **安全** | 密码明文存储 | 数据库泄露后用户密码直接暴露 |
| **安全** | Token 无过期机制 | Token 泄露后可永久使用 |
| **安全** | TCP 明文传输 | 敏感数据可被中间人窃听 |
| **功能** | 聊天为轮询而非推送 | 消息有最多3秒延迟，未能充分利用 Netty 长连接 |
| **功能** | 缺少评价/信用体系 | 无法建立买卖双方的信任机制 |
| **功能** | 搜索仅支持标题模糊匹配 | 无法按价格区间、发布时间等维度筛选 |
| **功能** | 缺少退货/退款流程 | 已完成订单无法逆向操作 |
| **工程** | 部分 Service 无单元测试 | 订单和消息模块缺少自动化验证 |
| **工程** | 客户端UI较简陋 | Swing 默认皮肤视觉效果一般 |
| **架构** | 单服务实例，无集群 | 无法应对高并发场景 |
